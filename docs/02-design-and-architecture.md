# Проектиране и архитектура на решението

## 1. Тип и структура на приложението

### Тип на приложението
ObreshkovLibrary е уеб базирано приложение за управление на училищна библиотека. То е разработено като ASP.NET Core MVC проект и се използва през браузър.

### Организация на проекта
Проектът е организиран по модел MVC (Model-View-Controller), който разделя логиката на приложението на отделни части:
- **Models** – съдържат класовете за данни и бизнес обектите;
- **Views** – съдържат потребителския интерфейс;
- **Controllers** – обработват заявките и свързват логиката с изгледите.

Освен основните MVC папки, проектът включва и:
- **Data** – класове за базата данни, DbContext, миграции и seed данни;
- **Areas/Identity** – функционалност за вход, роли и потребители;
- **wwwroot** – статични файлове като CSS, JavaScript и изображения;
- **ViewModels** – помощни модели за работа с форми и изгледи.

### Основни модули
Основните модули в системата са:
- модул за потребители и роли;
- модул за книги;
- модул за категории;
- модул за читатели;
- модул за заемане и връщане на книги;
- модул за любими книги;
- модул за известия;
- модул за публичен каталог;
- модул за ученически портал;
- модул за училищни новини.

## 2. Модел на данните

Системата използва база данни за съхранение на информацията. Основните ентитита и таблици в проекта са следните:

### Book
Представя книга в библиотеката.

**Основни полета:**
- Id
- Title
- Author
- Description
- CoverImageUrl / CoverPath
- CategoryId
- ISBN / допълнителни информационни полета
- наличност и информация, свързана с екземпляри

**Връзки:**
- една книга принадлежи към една категория;
- една книга може да участва в много заемания;
- една книга може да бъде добавена в любими от много ученици;
- една книга може да участва в известия за наличност.

### Category
Представя категория или подкатегория на книга.

**Основни полета:**
- Id
- Name
- ParentCategoryId

**Връзки:**
- една категория може да има много книги;
- една категория може да има подкатегории;
- една подкатегория принадлежи към една родителска категория.

### Reader
Представя читател в системата, най-често ученик.

**Основни полета:**
- Id
- FullName
- CardNumber
- Class
- PhoneNumber
- Email
- UserId (при връзка с потребителски акаунт)

**Връзки:**
- един читател може да има много заети книги;
- един читател може да има много известия;
- един читател може да има много любими книги.

### Loan
Представя заемане на книга.

**Основни полета:**
- Id
- ReaderId
- BookId
- LoanDate
- DueDate
- ReturnDate
- IsReturned
- полета, свързани с напомняния и просрочване

**Връзки:**
- едно заемане принадлежи на един читател;
- едно заемане е свързано с една книга.

### FavoriteBook
Представя връзка между ученик и любима книга.

**Основни полета:**
- Id
- ReaderId или UserId
- BookId
- CreatedOn

**Връзки:**
- един ученик може да има много любими книги;
- една книга може да бъде любима на много ученици.

### ReaderNotification
Представя съобщение или известие към ученик.

**Основни полета:**
- Id
- ReaderId
- Title
- Message
- IsRead
- CreatedOn

**Връзки:**
- един читател може да има много известия.

### SchoolNews
Представя новина, която се показва в началната страница.

**Основни полета:**
- Id
- Title
- Content
- PublishedOn
- ImageUrl

### Identity потребители и роли
Системата използва ASP.NET Core Identity за:
- вход в системата;
- управление на потребители;
- роли като администратор и ученик;
- ограничаване на достъпа до различните части на приложението.

## 3. Диаграма или структурирано описание

Вместо UML диаграма е използвано структурирано описание на основните класове и тяхната роля:

| Клас / Ентити | Роля в системата |
|---|---|
| Book | Съхранява информация за книгите в библиотеката |
| Category | Организира книгите по категории и подкатегории |
| Reader | Представя читател или ученик |
| Loan | Проследява заемането и връщането на книги |
| FavoriteBook | Пази любими книги на ученик |
| ReaderNotification | Съхранява известия към потребителите |
| SchoolNews | Пази новини за началната страница |
| ApplicationUser / IdentityUser | Представя потребител за вход в системата |

### Основни връзки между обектите
- Category → Book = едно към много
- Reader → Loan = едно към много
- Book → Loan = едно към много
- Reader → ReaderNotification = едно към много
- Reader ↔ Book = много към много чрез FavoriteBook
- Category → Category = йерархична връзка чрез ParentCategoryId

## 4. Потребителски поток

### Основни екрани / страници
Системата съдържа няколко основни групи страници:

#### Публична част
- начална страница;
- каталог;
- детайли за книга;
- страница с училищни новини;
- вход в системата.

#### Ученическа част
- мой профил;
- каталог за ученик;
- детайли за книга;
- любими книги;
- мои съобщения / известия;
- данни за текущо използване на библиотеката.

#### Административна част
- управление на книги;
- управление на категории;
- управление на читатели;
- управление на заемания;
- работа със съдържание и вътрешни административни екрани.

### Как потребителят преминава през системата

#### Гост
1. Отваря началната страница.
2. Разглежда каталога.
3. Отваря детайли за книга.
4. При нужда преминава към вход в системата.

#### Ученик
1. Влиза в системата.
2. Отваря своя профил.
3. Разглежда каталога.
4. Отваря детайли за книга.
5. Добавя книга в любими или следи известията си.
6. Проверява съобщения, свързани със заемания или наличност.

#### Администратор
1. Влиза в системата.
2. Отваря административния панел.
3. Управлява книги, категории и читатели.
4. Създава заемания и обработва връщания.
5. Следи данните в системата и актуализира съдържанието.

### Основни действия
- вход в системата;
- преглед на каталог;
- преглед на детайли за книга;
- добавяне и редактиране на книги;
- добавяне на книга в любими;
- създаване на заемане;
- връщане на книга;
- изпращане и преглед на известия.

## 5. Използвани технологии

### Езици
- C#
- HTML
- CSS
- JavaScript

### Framework и платформа
- ASP.NET Core MVC
- Razor Views

### База данни
- SQL Server

### Библиотеки и допълнителни технологии
- Entity Framework Core
- ASP.NET Core Identity
- Bootstrap

### Защо са избрани тези технологии
**C#** е подходящ език за разработка на стабилни приложения и е основен за обучението по специалността.

**ASP.NET Core MVC** позволява добра организация на проекта чрез разделяне на логиката, данните и потребителския интерфейс.

**Entity Framework Core** улеснява работата с базата данни чрез обектно-релационен модел и прави проекта по-лесен за поддръжка.

**SQL Server** е подходяща релационна база данни за съхранение на книги, читатели, заеми и потребителска информация.

**ASP.NET Core Identity** осигурява готова и надеждна система за вход, потребители и роли.

**Bootstrap**, HTML, CSS и JavaScript подпомагат изграждането на потребителския интерфейс и визуалното оформление на страниците.
